# Learning Git and GitHub

A comprehensive guide to mastering GitHub and Git proficiency: learn how to write excellent README files, master Markdown and GitHub Flavored Markdown (GFM), collaborate effectively with teams, use templates, and publish documentation with GitHub Pages.

## Quick Start

- **New to GitHub?** Start with [`docs/getting-started.md`](docs/getting-started.md) — step-by-step guide to creating and pushing your first repository.
- **Learn Markdown on GitHub** Read [`docs/markdown.md`](docs/markdown.md) — comprehensive guide to GFM syntax with practical examples.
- **Want to contribute?** See [`CONTRIBUTING.md`](CONTRIBUTING.md) for guidelines and workflow.

## What's Inside

### Documentation
- **`docs/getting-started.md`** — How to create a repo, push to GitHub, and enable GitHub Pages (PowerShell, `gh` CLI, and web UI steps)
- **`docs/markdown.md`** — Complete guide to Markdown and GitHub Flavored Markdown (GFM) with examples

### Community Files
- **`CONTRIBUTING.md`** — How to contribute to this project
- **`CODE_OF_CONDUCT.md`** — Community guidelines and expectations
- **`.github/ISSUE_TEMPLATE/`** — Templates for bug reports and feature requests
- **`.github/pull_request_template.md`** — Template for pull requests

### Templates & Examples
- **`templates/README-template.md`** — Reusable README template for your own projects
- **`examples/simple-project/README.md`** — Example of a minimal README

### Automation
- **`.github/workflows/ci.yml`** — CI workflow for markdown linting and link checking
- **`.markdownlint.json`** — Configuration for markdown style enforcement
- **`CODEOWNERS`** — Automatically request reviews from doc maintainers

## Topics Covered

- Writing clear, engaging README files
- GitHub Flavored Markdown (GFM) syntax and best practices
- GitHub collaboration features (issues, pull requests, discussions)
- Creating and using templates
- Publishing docs with GitHub Pages
- Setting up CI/CD for documentation
- Community and contribution guidelines

## Getting Started

1. Clone or fork this repository
2. Read [`docs/getting-started.md`](docs/getting-started.md) to learn the basics
3. Explore [`docs/markdown.md`](docs/markdown.md) for detailed Markdown examples
4. Check out the `examples/` folder for real-world README samples
